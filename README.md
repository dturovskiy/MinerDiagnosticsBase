# Miner Diagnostics Base

Це стартова база для діагностики руху, фізики та драбини в Unity-проекті.

## Що входить

- `Assets/Scripts/Diagnostics/Runtime/*` — код системи логування
- `Docs/QUICK_START_UA.md` — швидке підключення
- `Docs/FILES_AND_LOGS_UA.md` — які runtime-файли створюються
- `Docs/USAGE_PATTERNS_UA.md` — як логувати стани, драбину і помилки
- `Docs/BUG_REPORT_TEMPLATE_UA.md` — шаблон, який зручно надсилати разом з логами
- `Examples/LogsSample/*` — приклад згенерованих файлів

## Для чого це

Ця база вирішує три задачі:

1. Дає короткі структуровані події в `events.jsonl`
2. Зберігає зрозумілий `summary.txt`
3. Тримає поточний стан героя в HUD і в кожному записі логу

## Коли ця система активна

За замовчуванням логування активне тільки в:

- Unity Editor
- Development Build

У Release Build код не пише діагностичні файли.

## Основна ідея

Не логувати все щокадру в файл.  
У файл пишуться тільки важливі події і помилки.

Поточний стан героя оновлюється через `GameDiag.SetHeroSnapshot(...)`, а коли відбувається щось важливе — викликається `GameDiag.Event(...)`, `GameDiag.Warning(...)` або `GameDiag.Error(...)`.
